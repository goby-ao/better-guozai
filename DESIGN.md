# 果仔的一天 App 设计系统

> 正式 App 名称为“果仔的一天”。本文定义 iPad 优先、兼容 iPhone 的全年成长记录体验。

## 产品基准

- [Apple 提醒事项](https://apps.apple.com/us/app/reminders/id1108187841)：借鉴熟悉的清单层级、快速创建和低摩擦完成操作，不照搬系统应用外观。
- [Streaks](https://streaksapp.com/)：借鉴一项任务一步完成、周期目标和量化习惯，但不采用中断即失败的激励方式。
- [Finch](https://help.finchcare.com/hc/en-us/articles/37935669335309-Our-Approach-to-Self-Care)：借鉴温和鼓励、小步成长和成长领域组织方式，不引入虚拟货币经济。

## 方向摘要

- **视觉主张**：温暖的全年成长手账，以米白纸张、墨蓝文字和海水蓝、芒果黄、叶绿、珊瑚红构成克制而有活力的儿童成长界面。
- **内容计划**：今日页负责定向、显示状态和完成行动；成长页用成长花园与星图解释长期变化；勋章页承载荣誉与孩子自主选择的周心愿；家长中心负责计划、分析和数据管理。
- **交互主张**：手指直接划过任务形成实时手绘笔触并完成打卡；每次今日达成让花园成长一步；按钮只使用短促缩放和淡入淡出。

## 1. 视觉主题与氛围

视觉关键词是“温暖、自主、成长”。界面继承现有 HTML 的纸张与手写气质，但不再模拟整张 A4 页面，也不保留暑假限定文案。它应像一本会回应触摸的成长手账，而不是学校成绩系统或成人效率工具。

全年采用明亮暖色主题。深色模式不是首版目标，避免降低儿童阅读对比度和破坏纸张感。成长花园是长期成长的第一视觉锚点，任务划线是第一交互锚点。

App 图标暂定为暖米白底上的“芒果黄朝阳 + 海水蓝勾线”，让一天的开始、任务完成和成长星图共享同一视觉符号，不使用真人照片或固定卡通人物。

## 3. 字体规则

品牌文字是“亲切、有手感、能长时间阅读”。拒绝把通用几何无衬线字体作为展示字体，也不引入网络字体。

- 展示标题：优先使用 iOS 自带 `Kaiti SC`，以 `Font.custom(_:size:relativeTo:)` 接入 Dynamic Type；若设备不可用，则回退系统中文字体。
- 功能文字：系统字体，任务正文优先使用圆润设计但确保中文回退清晰。
- 数字：系统字体并使用等宽数字，适用于日期、完成量、连续天数和图表坐标。

| 层级 | iPad 基准 | iPhone 基准 | 字重与行高 |
| --- | ---: | ---: | --- |
| 页面标题 | 40pt | 34pt | 700，约 1.06 |
| 分区标题 | 28pt | 24pt | 700，约 1.15 |
| 任务正文 | 22pt | 18pt | 650，约 1.30 |
| 正文 | 18pt | 17pt | 400，约 1.45 |
| 辅助文字 | 14pt | 13pt | 600，约 1.30 |

所有文字支持 Dynamic Type。任务名称允许两行，第三行前应通过自适应布局扩大任务行，而不是截断关键信息。

## 4. 组件样式

### 任务行

- 采用整行列表结构而非一组独立卡片。
- 左侧打卡区最小 `52×52pt`，使用成长领域浅色底和高对比圆形复选框。
- 任务正文占主要宽度，完成量和标签置于次级区域。
- 行间使用细分隔线，不为每行添加阴影。
- 已完成状态保留手绘划线，文字降低对比度，复选框同步填充。

### 手指划掉交互

- `DragGesture` 仅在任务正文区域识别，避免与列表滚动争抢。
- 笔触跟随手指从左向右生长；达到任务文字宽度约 60% 后松手才完成。
- 达成阈值时提供一次轻触觉反馈，任务状态与复选框同时更新。
- 未达到阈值时笔触平滑淡出并收回，不使用弹跳。
- 完成后提供短时“撤销”；辅助功能、键盘和减少动态效果场景始终可使用复选框。

### 按钮

- 主按钮固定 `11pt` 圆角、海水深蓝实底、最小高度 `52pt`。
- 次按钮使用纸张底与明确的背景层级，不使用玻璃模糊。
- 按下状态仅缩放至 `0.96`，不使用弹性动画。
- 图标优先使用单一 SF Symbols 体系，图标按钮必须有辅助功能标签。

### 日期印章与勋章

- 日期印章使用双环圆形结构，延续 HTML 的视觉记忆。
- 系统勋章保持统一几何骨架，通过中心符号与领域色区分。
- 特别勋章允许家长选择颜色和 SF Symbol，但仍使用同一骨架，避免像随机贴纸集合。

### 图表

- 果仔视图使用成长星图，不出现失败率、排名或密集坐标轴。
- 家长趋势使用折线图；领域比较使用横向条形图或堆叠条；日期与领域矩阵使用热力格。
- 禁止六领域饼图和雷达图，它们难以准确比较。
- 图表数字使用等宽数字，标签左对齐，数值右对齐，网格线使用 `line` 的低对比版本。

## 5. 布局原则

采用 `4 / 8 / 12 / 18 / 24 / 32 / 48pt` 间距节奏。圆角尺度固定为 `7 / 11 / 18 / pill`，分别用于小标签、按钮与输入、大型内容区、胶囊状态。

- iPad：侧边栏负责四个主入口，内容区以可读宽度为先；今日页可在宽屏中让任务列表与今日状态并列，但任务始终是主列。
- iPhone：四个底部 Tab，每个 Tab 拥有独立导航历史；今日页单列，任务操作保持在拇指可达区域。
- 家长分析：顶部先给状态摘要，下面才是时间趋势和领域细节；筛选器靠近图表，不使用营销式大标题。
- 表单：使用完整页面或内联展开编辑长期模板，不把复杂任务配置塞入多层弹窗。

## 6. 深度与层级

以背景明度差和分隔线为主，阴影为辅：

- 层级 0：`canvasWarm`，应用背景与侧边栏。
- 层级 1：`paper`，今日任务和主要阅读表面，与背景至少保持可辨明度差。
- 层级 2：浮动编辑器、文件导入预览等临时聚焦表面，使用克制的系统阴影。

不使用玻璃拟态，不给每个信息块添加相同圆角和阴影，不使用紫蓝渐变。星图的发光只属于“今日达成”等具有语义的反馈。

## 7. 应做与避免

### 应做

- 每次打开先让果仔看懂今天做什么和已经完成多少。
- 使用真实的果仔任务、标签和成长数据作为预览样例。
- 将“跳过”呈现为中性状态，将“未完成”呈现为事实而非失败。
- 所有关键操作提供触控、辅助功能和键盘等价路径。
- 长文本、动态字体和横竖屏切换均保持任务内容完整。

### 避免

- 不做排行榜、红色失败率、扣分或连续天数归零动画。
- 不让果仔首页出现成人仪表盘或过多数字。
- 不用一屏相同卡片网格代替内容层级。
- 不用纯装饰动画；每个动画必须解释完成、获得或切换。
- 不让成长领域颜色成为识别信息的唯一手段。

## 8. 响应式与平台行为

- 最低支持 iOS 与 iPadOS 17。
- iPad 使用 `NavigationSplitView`，支持横竖屏、分屏和窗口尺寸变化，不假定全屏宽度。
- iPhone 使用 `TabView + NavigationStack`，紧凑宽度下将次级数据移入详情页，不缩小任务字体硬塞。
- 触控目标最低 `52pt`，高于系统通用下限，以适配儿童操作。
- 支持 Dynamic Type、VoiceOver、提高对比度和减少动态效果。
- 星图在紧凑宽度下从年视图降级为月视图入口，完整年视图仍可横向探索。

## 9. Agent Prompt Guide

### 快速色彩参考

`canvasWarm: oklch(0.94 0.025 91)`，`paper: oklch(0.99 0.012 92)`，`ink: oklch(0.30 0.045 230)`，`ocean: oklch(0.55 0.105 216)`，`oceanDeep: oklch(0.39 0.09 220)`，`mango: oklch(0.83 0.15 82)`，`leaf: oklch(0.63 0.105 155)`，`coral: oklch(0.69 0.13 34)`。

### 页面提示词

1. 创建 iPad 今日页：`canvasWarm` 侧边栏，`paper` 主内容面，页面标题 40pt、字重 700，任务文字 22pt、行高 1.30，任务行无卡片阴影、仅用细分隔线，左侧打卡区域至少 52pt，按钮圆角 11pt。
2. 创建任务划线交互：在任务文字区域用 `DragGesture` 跟踪从左到右的手指路径，笔触颜色使用该任务成长领域主色，超过文字宽度 60% 后完成，触发轻触觉反馈并同步复选框，未达阈值使用 160ms ease-out 淡出，无弹跳，减少动态效果时直接切换删除线。
3. 创建成长星图：年视图显示 365 个稳定日期位置，星体亮度表示必做完成比例，`mango` 光环表示今日达成；月视图将每天展开成六领域形状；空白日期使用 `line` 浅色且不标记失败，点击进入日详情。
4. 创建家长分析页：顶部为本周状态摘要，下面用折线图展示每日完成趋势、横向条形图比较成长领域、矩阵展示日期与领域；标题 28pt、正文 18pt、图表数字等宽，网格线为 `line` 低对比色，不使用饼图、雷达图或排行榜。
5. 创建勋章馆：系统勋章与特别勋章使用同一圆形几何骨架，卡片半径 18pt，系统勋章标注规则来源，特别勋章显示家长寄语；已获得勋章使用实色，未获得勋章使用轮廓和清晰条件文字，不使用锁链或灰暗惩罚图标。
