# 果仔的一天实施计划

## 1. 目标与边界

“果仔的一天”是一个 iPad 优先、兼容 iPhone 的全年成长 Todo App。果仔负责查看今日计划、自主打卡、自评和发起“我的挑战”；家长负责长期模板、历史回顾、勋章、奖励、数据分析与备份恢复。

首版边界：

- 单个果仔档案，无账号、无服务端。
- 最低支持 iOS 与 iPadOS 17。
- SwiftData 本地持久化，手动导出到系统“文件”或 iCloud Drive。
- 不做 CloudKit 自动同步、积分商城、排行榜、照片或语音记录。
- 代码批准后创建在 `/Users/yao/projects/better-guozai/GuozaiDay/`。

领域语言以 [CONTEXT.md](../CONTEXT.md) 为准，视觉实现以 [DESIGN.md](../DESIGN.md) 为准。

## 2. 核心用户流程

### 果仔

1. 打开“今日”，看到必做任务、选做任务和当前进度。
2. 点击复选框或用手指划过任务完成打卡。
3. 量化任务可选填写实际完成量。
4. 可以添加当天的选做“我的挑战”。
5. 完成后填写心情、自评和最满意的一件事。
6. 查看成长星图、新勋章和已解锁的心愿奖励。

### 家长

1. 创建带重复规则、成长领域、标签和可选量化目标的任务模板。
2. 查看或调整某日计划，不影响历史快照。
3. 回顾历史、修正补记、留下鼓励。
4. 查看趋势、领域分布、完成量以及跳过和未完成情况。
5. 管理心愿奖励、颁发特别勋章。
6. 导出、导入、恢复或清空数据。

## 3. 信息架构

四个主入口：

- **今日**：日期状态、必做与选做任务、我的挑战、每日回顾。
- **成长**：年/月/日成长星图、本周亮点、趋势详情。
- **勋章**：系统勋章、特别勋章、心愿奖励。
- **家长中心**：任务模板、详细分析、历史修正、提醒、导入导出与设置。

iPad 使用 `NavigationSplitView`，iPhone 使用 `TabView + NavigationStack`，每个 Tab 保留独立导航历史。

## 4. 技术架构

### 平台与框架

- Swift 6 语言模式按 Xcode 16.4 可稳定编译的能力使用。
- SwiftUI 构建自适应 UI。
- SwiftData 提供本地存储与迁移基础。
- Swift Charts 构建家长趋势图。
- UserNotifications 提供本地计划提醒和可选任务提醒。
- FileDocument、fileExporter、fileImporter 提供明文 JSON、CSV 备份恢复。
- SF Symbols 作为唯一图标体系。

### 分层

```text
App Shell
  ├─ Today Feature
  ├─ Growth Feature
  ├─ Badges Feature
  └─ Parent Feature
        ↓
Domain Services
  ├─ DailyPlanGenerator
  ├─ CheckInService
  ├─ AchievementEngine
  ├─ AnalyticsService
  ├─ NotificationScheduler
  └─ BackupService
        ↓
SwiftData Store / JSON + CSV Documents
```

原则：

- 一个根 `ModelContainer`，避免多个页面产生不同数据源。
- 简单界面状态使用 `@State`、`@Binding`，共享服务通过 `@Environment` 注入。
- 不为每个页面默认创建 ViewModel；复杂业务规则放入可单元测试的纯服务。
- 历史记录只追加或显式修正，任务模板修改不追溯更新历史快照。

## 6. 数据模型草案

### `ProfileRecord`

- `id`、昵称、可选头像、可选当前年级。
- 首版仅一个档案，但所有业务记录保留 `profileId`。

### `TaskTemplateRecord`

- 标题、成长领域、标签、必做或选做。
- 重复规则：每天、工作日、周末、自选星期。
- 开始与结束日期、暂停区间。
- 可选目标值、单位和单独提醒时间。

### `DailyPlanRecord`

- `dayKey`，使用本地自然日标识。
- 生成时间、最后修改时间、所属档案。
- 任务模板只负责生成，计划本身是独立历史快照。

### `DailyTaskRecord`

- 当日标题、成长领域、标签快照。
- 来源：任务模板、家长单次任务、我的挑战。
- 必做或选做。
- 状态：待完成、已完成、已跳过；过去日期的待完成显示为未完成。
- 目标值、单位、实际完成量。
- 完成时间、跳过原因、修正时间。

### `DailyReflectionRecord`

- 心情、自评星级、最满意的一件事、家长鼓励。

### `BadgeAwardRecord`

- 勋章稳定标识、名称、来源、获得时间。
- 系统勋章保存规则版本和证据记录 ID。
- 特别勋章保存家长填写的理由。
- 历史修正可以补发符合条件的勋章，但不撤销已获得勋章。

### `WishRewardRecord`

- 名称、关联目标或勋章、解锁时间、领取状态。

### `TagRecord`

- 自定义标签名称和稳定 ID；成长领域使用固定枚举，不与标签混用。

## 7. 关键业务规则

### 每日计划生成

- 每天首次读取时按任务模板生成当天快照。
- 同一模板同一天只生成一次，使用稳定组合键防重。
- 当日临时调整不修改未来模板。
- 删除模板只停止未来生成，暂停模板不删除模板和历史。

### 今日达成

- 全部必做任务已完成时为“今日达成”。
- 选做任务不阻塞达成。
- 跳过不冒充完成，未完成不使用惩罚文案。

### 历史修正

- 果仔只操作今天，家长可补记或修改过去日期。
- 同时保存实际发生日期和修正时间。
- 修正后重新计算统计，并只补发、不撤销勋章。

## 8. 勋章与奖励

### 系统勋章组

- 起步：首次打卡、首次今日达成、首次我的挑战。
- 坚持：累计达成、7 天内达成 5 天等弹性目标。
- 探索：首次覆盖新成长领域、六领域均有记录。
- 自主：主动创建并完成我的挑战。
- 回归：中断后重新开始。
- 分类成长：阅读、运动、生活自理等累计里程碑。

规则必须可解释、版本化、可单元测试。记录当前连续天数和历史最长天数，但不让勋章依赖绝对不断档。

### 特别勋章与心愿奖励

- 家长可手动颁发特别勋章并填写理由。
- 家长可设置与周目标或勋章关联的一次性心愿奖励。
- 周目标心愿由果仔自主选择一个作为当前心愿，按周一至周日统计，今日达成 5 天后解锁，每周最多解锁一个。
- 未达成的当前心愿可延续到下周；已经解锁的心愿不会因后续撤销或历史修正自动收回。
- 成长页提供非消费型成长花园，每个今日达成成长一步，28 步长成一棵永久保留的树。
- 不设置金币、积分消费、扣分、勋章降级和排行榜。

## 9. 数据可视化

### 果仔视角

- 年视图：365 颗星，亮度表示必做完成比例，金色光环表示今日达成。
- 月视图：每日展开为六领域成长星。
- 日视图：任务、完成量、回顾、勋章和家长鼓励。
- 本周亮点采用客观、鼓励性的规则生成，不显示失败率。

### 家长视角

- 折线图：每日完成趋势、量化任务趋势。
- 横向条形图：成长领域比较。
- 日期与领域矩阵：习惯节奏。
- 周期对比：只与果仔自己的上一周期比较，不做同龄排名。
- 跳过、未完成及其原因分开统计。

## 10. 数据导出与恢复

### 完整备份 JSON

文件包含：

- `schemaVersion`、`exportedAt`、App 版本。
- 成长档案、标签、模板、每日计划与任务。
- 回顾、勋章、心愿奖励和规则版本。
- 日期使用 ISO 8601，稳定对象使用 UUID。

### 分析 CSV

- `daily_summary.csv`
- `task_records.csv`
- `quantity_records.csv`
- `badge_awards.csv`

### 安全导入

1. 校验文件类型、版本和必需字段。
2. 预览日期范围、任务数、打卡数和勋章数。
3. 默认按稳定 ID 合并并跳过重复项。
4. 冲突记录由家长选择，不静默覆盖。
5. 清空后恢复是独立操作，二次确认并先提示导出备份。

首版只有一个成长档案：导入时把备份归入本机唯一档案；仅当本机仍是未使用的初始示例时，才在确认导入后替换这批示例，避免新设备恢复产生第二个档案或重复模板。

## 11. 提醒与辅助功能

- 每日开始提醒和晚间回顾提醒，时间可调整或关闭。
- 单独任务提醒可选，默认关闭。
- 单项提醒滚动安排未来 6 次，并在打开今日页时续排；提醒日期与任务生成共用开始、结束、星期和暂停规则。
- 任务正文 iPad 约 22pt、iPhone 约 18pt，并支持 Dynamic Type。
- 触控区域至少 52pt。
- 支持 VoiceOver、提高对比度、减少动态效果和键盘操作。
- 图表同时使用颜色、形状、亮度与文字。

## 12. 标志性交互

任务可点击复选框，也可用手指从左向右划过文字完成：

- 笔触实时跟随手指。
- 超过文字宽度约 60% 后松手才完成。
- 达成阈值时提供轻触觉反馈。
- 未达阈值时平滑收回，不弹跳。
- 完成后提供短时撤销。
- 减少动态效果时直接显示普通删除线。

实现时重点验证手势与纵向滚动、iPad 指针、动态字体和 VoiceOver 不冲突。

## 14. 验证策略

优先使用低成本、可重复验证，避免无必要的 Xcode Build 或模拟器测试：

- 单元测试：重复规则、每日防重生成、达成判断、状态转换、勋章规则、统计聚合、导入合并。
- 往返测试：导出后重新导入，关键记录逐字段一致。
- SwiftUI Preview：iPad 与 iPhone、空数据、长文本、大字体、完成和跳过状态。
- 必要构建：项目骨架完成后一次、关键模块合并后一次、最终交付前一次。
- 必要运行验证：任务划线手势、文件选择器、通知权限、辅助功能等无法仅靠单元测试覆盖的流程。
